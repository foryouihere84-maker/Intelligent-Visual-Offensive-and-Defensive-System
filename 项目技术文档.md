# AI模型管理与对抗攻击系统技术文档

## 项目概述

本项目是一个功能完整的AI模型管理与对抗攻击Web应用系统，基于Flask框架构建。系统集成了目标检测、图像分类、模型训练、对抗攻击等多种AI功能，提供友好的Web界面供用户操作。

系统主要包含四大核心功能模块：
- **目标检测**: 基于YOLOv8的目标检测功能
- **对抗攻击**: 包含数字域和物理世界的多种对抗攻击算法
- **模型训练**: 支持YOLOv8模型的增量训练和微调
- **模型管理**: 训练模型的统一管理和维护

**当前开发状态**: 项目已完成全部核心功能开发，具备完整的前后端交互、模型管理、训练流水线和对抗攻击能力，可以直接投入实际使用。

## 系统特点

### 核心优势
- **多功能集成**: 集成目标检测、对抗攻击、模型训练于一体
- **算法丰富**: 支持10+种对抗攻击算法，涵盖FGSM、PGD、CW等经典方法
- **物理攻击**: 独创AdvCam和AdvShadow物理世界对抗攻击
- **智能训练**: 支持YOLOv8增量训练，自动参数优化

### 技术特色
- **动态模型发现**: 自动扫描和加载本地模型文件
- **单例模型管理**: 全局模型缓存，避免重复加载
- **多类型支持**: 支持YOLO(.pt)和PyTorch(.pth)模型格式
- **实时响应**: 异步处理机制，流畅的用户体验

## 数据流设计

### 请求处理流程
1. **用户请求** → Flask路由分发到对应蓝图
2. **文件上传** → 安全验证和存储
3. **参数解析** → 表单数据和配置参数提取
4. **模型获取** → 通过ModelManager加载所需模型
5. **业务处理** → 调用核心处理器执行具体任务
6. **结果保存** → 处理结果存储到results目录
7. **响应封装** → JSON格式或HTML模板渲染
8. **前端展示** → 结果可视化呈现

### 模型管理机制
```
ModelManager (单例模式)
├── 模型缓存 (_models字典)
├── 动态配置刷新 (_refresh_models_config)
├── YOLO模型加载 (支持.pt格式)
├── TorchVision模型加载 (支持.pth格式)
├── 预训练模型支持
└── 内存优化管理
```

### 文件处理流程
```
上传文件
├── 安全验证 (文件类型、大小限制)
├── 重命名处理 (防止冲突)
├── 临时存储 (static/uploads/)
├── 业务处理 (调用处理器)
├── 结果保存 (static/results/)
└── 清理机制 (训练后清理临时文件)
```
